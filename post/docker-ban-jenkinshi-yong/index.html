Docker 版 Jenkins使用

2026-05-11 · 1 分钟阅读

Jenkins 是一个非常强大的软件方便部署，之前很少用到，现在我要区分开发环境和测试环境，也只需要用到很少一部分 Jenkins 功能。
先把 vue 的前端项目从 gitee 上拉取过来（需要安装 gitee 插件）自动打包好（需要有 node 环境），然后通过 publish over ssh 部署到物理机上，用 nginx 做代理（在 Jenkins docker 做代理非常的麻烦）。

安装 Gitee Plugin

Jenkins 这块真的让人迷惑，gitee 插件配置好了，但是后面拉取代码还是要用公钥或者是账户密码的方式，插件只是用来接收 gitee webhook 通知这些的。

nodeJs 环境配置

安装 NodeJS Plugin插件

publish on ssh

安装 Publish Over SSH插件，配置过程 系统管理 -> 系统配置 -> Publish over SSH

  1. hostname的话 Docker 有专门一个自定义的域名指向物理机
  2. Remote Directory 在 Windows 下根目录只能是当前登录用户下的用户目录，这点要注意，不然找不到文件了。
